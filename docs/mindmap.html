🔄 API 请求处理流程

sequenceDiagram participant C as 客户端 participant W as Cloudflare Worker participant A as 认证中间件 participant T as Token 调度器 participant P as 代理池 participant G as Grok API C->>W: POST /v1/chat/completions W->>A: 验证 API Key A-->>W: 认证通过 W->>T: selectBestToken(model) T-->>W: 返回最优 Token (SSO) W->>P: getProxyForSso(token) alt 已绑定且健康 P-->>W: 返回已绑定的代理 else 未绑定 P->>P: Round-Robin 选择健康代理 P->>P: 自动绑定 SSO → 代理 P-->>W: 返回新分配的代理 end W->>G: 通过代理发送请求 alt 请求成功 G-->>W: 返回 Grok 响应 (NDJSON) W->>P: markSuccess(proxy) W-->>C: 转换为 OpenAI SSE 格式返回 else 请求失败 G-->>W: 返回错误 W->>P: markFailure(proxy) alt 累计失败 ≥ 3 P->>P: 标记代理不健康 P->>P: 解绑所有 SSO end W->>T: recordTokenFailure + cooldown W->>W: 重试 (最多3次) end

📡 代理管理 API 一览

端点方法功能说明
/api/v1/admin/proxiesGET查看代理池返回所有代理状态、健康、绑定关系
/api/v1/admin/proxiesPOST添加代理支持 {url} 单个 或 {urls:[]} 批量
/api/v1/admin/proxiesDELETE删除代理传入 {url} 删除指定代理
/api/v1/admin/proxies/assignPOST绑定 SSO{sso, proxy_url} 手动绑定
/api/v1/admin/proxies/unassignPOST解绑 SSO{sso} 解除绑定
/api/v1/admin/proxies/health/resetPOST重置健康所有代理恢复为健康状态
/api/v1/admin/proxies/testPOST测试代理{url} 检测代理可用性
/api/v1/admin/proxies/clearPOST清空代理池删除所有代理和绑定关系
